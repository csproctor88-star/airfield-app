GLIDEPATH

Glidepath Video Walkthrough — Production & Hosting Plan

Reference for recording, producing, and publishing Glidepath training walkthroughs.

Created 2026-06-07 · Tuned to: Shure MV7+ mic, 3200×2000 HiDPI panel (16:10), NVIDIA GPU (NVENC), OBS Studio.

1. Goal & Priority

Produce a library of screen-recorded walkthroughs, voiceover only (no facecam), narrated live while clicking. One set of recordings, repurposed four ways — in priority order:

  1. New-user onboarding course (primary) — sequenced chapters that walk a brand-new Airfield Manager through the whole app.
  2. Admin / base-setup guide — the harder admin tasks, aimed at base admins.
  3. In-app contextual help — the same onboarding chapters, re-surfaced individually inside /help/[module-id] at the moment of need.
  4. Marketing / sales reel — a 2–3 min supercut of the best moments for glidepathops.com.

Because the onboarding chapters are modular, recording them once feeds all four uses.

3. Audio Recipe (Shure MV7+)

Mic technique & MOTIV app

  • Connect via USB-C; 48 kHz.
  • Distance ~3–4 inches (a fist), speaking on-axis into the front face (logo side) — dynamic mic, so proximity = warmth and off-axis = muffled.
  • MOTIV Tone → Natural (Dark sounds muffled; Bright sounds airy).
  • High-Pass Filter ~80 Hz.
  • Run only one denoiser — MOTIV's or OBS's, never both (doubling thins/dulls the voice).
  • Set gain so normal speech peaks −12 to −6 dB (with filters off) before tuning the chain.

OBS mic filter chain (Audio Mixer → MV7+ gear → Filters), in this order:

  1. Noise Suppression → RNNoise (skip if MOTIV's denoiser is on)
  2. Noise Gate — Close −40 dB, Open −32 dB, Attack 10 ms, Hold 150 ms, Release 120 ms (this is what silences keyboard/mouse clicks in pauses)
  3. Compressor — Ratio 3:1, Threshold −18 dB, Attack 6 ms, Release 60 ms, Output Gain +3 dB
  4. Limiter — Threshold −3 dB, Release 60 ms
  5. (optional, for clarity) 3-Band Equalizer — High +3 dB, Mid −2 to −3 dB (cuts mud), Low 0

Tuning: clicks leaking through the gate → raise Open Threshold toward −28. Words clipped at the start → lower it. "Airy/thin" → get closer + Tone:Natural. "Muffled" → EQ high band up, back off distance slightly, ease denoiser.

4. Recording Workflow (record-and-ship)

  • Record against the Demo base — /login?demo=true (isolated Demo AFB seed). Fully populated, consistent across takes, and exposes no real base data or personnel names.
  • Browser in fullscreen (F11) — no tabs/address bar in frame.
  • Light bullet outline per chapter (not a word-for-word script) — keeps narration on-rails and takes short.
  • One take per chapter; if you fumble, pause and restate the sentence, trim later. Light edit only.
  • Optional reusable 3–5 sec title card per chapter.
  • 30-sec test before each session: read a sentence, type/click during a pause → confirm sharp text + clicks gated out. Check color in Chrome/VLC, not WMP.

6. Hosting & Delivery Plan

Primary host: YouTube (Unlisted)

  • Upload as Unlisted, organized into Playlists ("Glidepath Onboarding", "Glidepath Admin").
  • Embed via youtube-nocookie.com (privacy-enhanced — no tracking cookies until play; cleaner for gov use).
  • Captions / 508: rely on YouTube auto-captions, then correct in YT Studio. Near-zero-effort 508 path.

Why YouTube over self-hosting (Supabase/Vercel)

Storage isn't the problem — egress (bandwidth) is. Supabase Pro includes ~100 GB storage (the whole library is ~5–6 GB, trivial) but only ~250 GB egress/month, and video streaming burns egress fast (one 190 MB chapter × 100 views ≈ 19 GB). YouTube serves bandwidth free and at every resolution/device automatically. YouTube usually works on AFNet; for the rare blocked network, users watch on a personal device.

In-app surfacing (future — separate spec, not built yet)

  • /help/[module-id] pages and the onboarding flow embed the matching YouTube iframe.
  • On networks where YouTube is blocked: show a fallback message + link/QR to "watch on your phone."

Platform One contingency

External YouTube embeds won't be reachable inside an IL4/IL5 boundary. Archive the master .mp4s (~5–6 GB delivery / ~10–30 GB OBS masters) locally + a cloud backup, so the videos can be self-hosted if Glidepath migrates to Platform One.
